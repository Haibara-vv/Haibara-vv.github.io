简介：
    经过这么长时间的前端学习，现在也在js和jq的学习了，但在最近的项目里常常很多的CSS样式不记得了，于是打算花一晚上总结一下常会用到的CSS样式。虽然W3C School已经都有了，但是想着自己总结一遍印象会更深刻。如下：
ps: 篇幅有限，上一篇只总结了各类标签,本篇总结了CSS样式以及CSS选择器和浮动

选择器的优先级：
作用范围越大，优先级越低
类型选择器 < 类型选择器 < id选择器

类选择器(class选择器):
在一个页面中可以多次出现
1:给指定的标签添加属性class=”name1”
2:当成选择器使用时添加. .name1{}
一个class属性可以有多个值
class=”name1 name2 name3”

id选择器:
在一个页面相同的id值只能出现次，具有唯一性
1:给指定的标签添加属性id=”name1”
2:当成选择器使用时添加# #name1{}

  • 通配符 匹配页面中所有的元素
    *{margin:0;padding:0;}

群组选择器：选择器,选择器,选择器{}
包含选择器（后代选择器）:选择器 选择器{}
伪类选择器
:link{} 初始状态
:visted{}访问过后的状态
:hover{}鼠标悬停的状态
:active{}鼠标按下的状态
注意：按照以上的顺序书写

选择器:hover{} 移入中间，改变自己
选择器:hover 选择器{} 移入父级，改变子级

包含选择器：选择器权重之和
特殊样式设置：
改变A标签的文本颜色，选择器必须选择a

浮动相关概括：

float: 浮动;
作用：让元素横排展示（必须给所有需要横排展示的元素都加浮动）
left:左浮动
right:右浮动
none:不浮动

规则： 贴靠象限
    一行容纳不下，则换行显示，如果最终还是放不下，则凑合带着
    只占据自己内容位置大小的元素，添加浮动后（none除外）能设置宽高
    添加浮动的元素脱离文档流（正常文档流 从上到下 从左到右）

图文混排：
    图片添加浮动，文字环绕图片

清除浮动：
清除浮动带来的问题
问题：
当父级元素没有高度时，子元素浮动，父级元素高度塌陷，引起布局混乱

解决方式1：
给父级元素加高度；不灵活

解决方式2：清除浮动
    在浮动元素最后添加一个独占一行的空元素，空元素的样式威
    clear:left/right/both
    不断的添加空元素，造成代码冗余
