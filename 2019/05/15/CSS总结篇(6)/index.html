简介：
    经过这么长时间的前端学习，现在也在js和jq的学习了，但在最近的项目里常常很多的CSS样式不记得了，于是打算花一晚上总结一下常会用到的CSS样式。虽然W3C School已经都有了，但是想着自己总结一遍印象会更深刻。如下：
ps: 篇幅有限，本篇总结了CSS的一些核心属性(再续上篇)，本篇包括：关于最小高度引发的兼容问题、表单标签及属性高级等。

关于最小高度引发的兼容问题：

问题 bug
解决办法    hack

bug1:    img底部默认3px间距
hack:    给img添加display:block;

bug2:    行级元素和块级元素换行会解析一个空格
hack:    添加浮动

bug3:    浮动元素添加外边距，在ie6下产生双倍边距
hack:    给浮动元素添加display:inline;

bug4:    img在ie浏览器下默认有蓝色边框
hack:    给img添加border:0;

bug5:    input表单元素顶部对齐不一致
hack:    给input表 单元素添加float:;

bug6:    表单按钮的大小不一-致
hack:    通过css样式修改大小
        通过css样式修改大小
        如果需要用图片设置按钮，则添加背景图

bug7: 浮动元素宽度是50%,ie6下将将四舍五入,将大于100%
hack7:给 右浮动的元素添加clear:right;

bug8:ie9-支持hand,其他的不支持
hack8: cursor:pointer;

bug10:子元素垂直方向外边距溢出
hack10:给父元素添加overflow: hidden;
        给父元素添加paddig top:;
        给父元素添加border-top:;
        给子元素添加float:;

bug11:子元素是块级元素，学动有可能造成阶梯状显示
hack11:子元素和父元素都添加float:;

bug12: input有 默认的边框
hack12: border :none;

bug13:input中placeholder=""在ie6-不支持
hack13:使用value=""代替

bug1:    ie6-不识别min-height:;
hack:    height:auto !important;
        height:100px;
        min-height:200px;

一屏页面书写:

html,body{
    height:100%;
    width:100%;
}

表单标签及属性高级：

fieldset > legend:表单字段集
    一个filedset里面只允许出现一个legend，相当于是标题

label 关联提示信息标签
    for="id值"
    <标签 id="值"></标签>


input type="fiel" 文件上传域
    multiple="可选多个文件"    前提是按住ctrl

表格高级

th:相当于是td单元格
    文本内容上下居中，左右居中加粗
thead:    表头
tbody:    表体
tfoot:    表尾

border-spaceing:边框与边框之间的间隙;不能 为负值

border-collapse:边框是否合并;
    separate    默认值
    collapse    合并为单边框表格

empty-cells:    单元格没有内容时的设置;
    show        显示
    hide        隐藏

table-layout:    单元格和行的算法;
    auto        默认值（自适应）
    fixed        固定值    高度还是会随内容变化

caption:    表格标题标签    
    caption-side:left/right/    火狐支持
        top/            ie7支持
        bottom  ;        都支持

rules="分割线"
    rows    行分割线
    cols    列分割线
    all        行和列的分割线
    none    没有分割线
    groups    列组分割线

col 和 colgroup    :  分列组
            span="多少列为一组"
            col的列组线只显示最后一列

bfc区域：是一个特殊的容器

格式化上下文
bfc的触发条件：
1.html（根元素）
2.float不为none
3.poistion:absolute/fixed;
4.display:
    inline-block 
    table-cell
    table-caption
    flex
    inline-flex 
5.overflow不为visible
6:计算bfc区域的高度时，浮动元素也要参与计算（清除浮动）

bfc区域的渲染规则

在一个BFC区域中，块级元素从上至下挨个排列
同属于同一个bfc区域中的元素，上下外边距相遇会发生重叠
子元素的左（右）外边距，将挨着包含元素的左（右）边框
bfc区域不会与float元素重叠（position）
bfc区域是一个独立的隔离容器，里面的子元素不会影响到外边的元素布局
    （两列布局，三列布局）
